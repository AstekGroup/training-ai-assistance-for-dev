# Modules de Cours

Ce dossier contient les modules de la formation **Vibe Coding** (parcours **3 jours**), découpés en **6 demi-journées** (2 modules par jour).

## Méthode en 6 modules


| #   | Jour          | Fichier                                               | Intitulé                                                                               |
| --- | ------------- | ----------------------------------------------------- | -------------------------------------------------------------------------------------- |
| 1   | J1 matin      | `MODULE_1_Structurer-Vibe-Coding-Prompt.md`           | Structurer le Vibe Coding et le prompt engineering en contexte entreprise              |
| 2   | J1 après-midi | `MODULE_2_Intensifier-Pratique-Outils-Git.md`         | Intensifier la pratique — outillage, Git d’équipe, labs et garde-fous                  |
| 3   | J2 matin      | `MODULE_3_Concevoir-stack-agent-MCP.md`               | Concevoir la stack agent — anatomie, Rules, Skills et premières intégrations MCP       |
| 4   | J2 après-midi | `MODULE_4_Industrialiser-agents-MCP-securite-BMAD.md` | Industrialiser les agents — MCP avancé, sécurité opérationnelle et méthode BMAD        |
| 5   | J3 matin      | `MODULE_5_Cadrer-developpement-harness-qualite.md`    | Cadrer le développement assisté — harness, contrat de contexte et qualité sur le cycle |
| 6   | J3 après-midi | `MODULE_6_Pratiquer-methodes-travail-adaptees.md`     | Pratiquer des méthodes de travail adaptées                                             |


Chaque fichier de module contient : objectifs, **planning** de demi-journée, **contenu détaillé** et lien vers le module suivant.

## Durée indicative

- **Par demi-journée** : ~3 h à 3 h 30 de séquences pédagogiques (+ pauses et déjeuner selon agenda réel).
- **Total** : 3 jours (agenda type **7 h** de présence par jour, conformément au syllabus).

## Public cible

Développeurs expérimentés, architectes, tech leads avec projet d’entreprise.
